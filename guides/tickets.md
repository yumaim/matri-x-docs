# 開発チケットの投稿方法

**バグ報告・機能リクエストの方法**

---

## 🎫 開発チケットとは

Matri-Xの改善提案やバグ報告をするための機能です。

あなたの声が、Matri-Xをより良くします！

<figure><img src="images/tickets-page.png" alt="開発チケットページ"><figcaption>開発チケットページ — 機能リクエストやバグ報告をチケットとして投稿できます</figcaption></figure>

---

## 📝 投稿方法

### 1. 開発チケットにアクセス

ダッシュボード → 「開発チケット」をクリック

### 2. カテゴリを選択

投稿したいカテゴリを選択:
- **💡 機能リクエスト** — 新機能の提案
- **🐛 バグ報告** — 不具合の報告
- **📚 ドキュメント改善** — ドキュメントの誤字・改善提案
- **💬 その他** — その他のフィードバック

### 3. チケットを作成

1. 「新規チケット」をクリック
2. タイトルを入力（簡潔に）
3. 詳細を入力（具体的に）
4. 「送信」をクリック

---

## ✅ 良いチケットの書き方

### 🐛 バグ報告

**必須情報:**
- 何をしたか（再現手順）
- 何が起きたか（実際の結果）
- 何が起きるべきか（期待する結果）
- 環境情報（ブラウザ、OS、画面サイズ）

**例:**

```
タイトル: フォーラム投稿時にエラーが発生する

詳細:
1. フォーラム → 新規投稿をクリック
2. タイトルと本文を入力
3. 「投稿」をクリック
4. → エラー「Something went wrong」が表示される

期待する結果: 投稿が正常に完了する

環境:
- ブラウザ: Chrome 120
- OS: macOS 14.2
- 画面サイズ: 1920x1080
```

**ヒント:**
- スクリーンショットがあると理解しやすい
- コンソールエラー（F12で確認）があれば添付

---

### 💡 機能リクエスト

**必須情報:**
- なぜその機能が必要か（背景・課題）
- どう使いたいか（具体的なユースケース）
- 代替案があるか

**例:**

```
タイトル: フォーラム投稿の下書き保存機能

詳細:
【背景】
長文を書いている途中で、誤って閉じてしまうことがある。

【提案する機能】
- 投稿作成中に、自動で下書き保存
- 次回アクセス時に、下書きから再開できる

【ユースケース】
1. 長文の事例共有を書いている
2. 急な用事でブラウザを閉じる
3. 後で戻ってきても、続きから書ける

【代替案】
- メモ帳で書いてからコピペ（面倒）
- こまめにコピーしておく（忘れる）
```

---

### 📚 ドキュメント改善

**必須情報:**
- どのページか（URL）
- どこが問題か（具体的に）
- どう改善すべきか（提案）

**例:**

```
タイトル: FAQ「登録にクレジットカードは必要ですか？」の記載が不明瞭

詳細:
URL: https://blog.matri-x-algo.wiki/faq

現在の記載:
> A. 不要です。完全無料で利用できます。

問題点:
「将来的に有料プランが出たときはどうなるのか」が不明

提案:
> A. 不要です。現在は完全無料で利用でき、将来的に有料プランを
> 追加する予定ですが、既存機能は無料のまま利用可能です。
```

## 🔔 チケットの通知

チケットの更新通知を受け取るには:

1. ダッシュボード → 「設定」
2. 「通知設定」
3. 「チケット通知」をON

以下の通知が届きます:
- 自分のチケットへの返信
- ステータス変更（対応中・完了など）

---

## 📊 チケットのステータス

| ステータス | 説明 |
|------------|------|
| **未対応** | 新規チケット、確認待ち |
| **対応中** | 開発チームが対応中 |
| **完了** | 対応完了・機能実装済み |
| **保留** | 将来的に対応予定 |
| **却下** | 対応しない（理由を記載） |

---

## 💬 返信が来たら

開発チームから返信が来ることがあります:

- 追加情報のお願い
- 進捗報告
- 対応完了の通知

通知を受け取ったら、できるだけ早く返信しましょう。
